## Installation on WordPress

### Method 1: Insert Headers and Footers Plugin (Recommended)

1. Install "Insert Headers and Footers" plugin
2. Go to Settings > Insert Headers and Footers
3. Paste this code in **Footer** section:

```html
<script>
  window.FFPMA_CHAT_API = 'http://130.49.160.73:3500';
  
  (function() {
    const link = document.createElement('link');
    link.rel = 'stylesheet';
    link.href = window.FFPMA_CHAT_API + '/widget.css';
    document.head.appendChild(link);
    
    const script = document.createElement('script');
    script.src = window.FFPMA_CHAT_API + '/widget.js';
    script.async = true;
    document.body.appendChild(script);
  })();
</script>
```

4. Save and test at forgottenformula.com

### Method 2: Theme Footer

1. Go to Appearance > Theme File Editor
2. Select footer.php
3. Paste the same code above before `</body>`
4. Save

### Method 3: Custom HTML Widget

1. Go to Appearance > Widgets
2. Add "Custom HTML" widget to Footer
3. Paste the same code
4. Save

## Server Management

**Current Process:**
- Running in background on VPS
- PID: Check with `ps aux | grep "node server.js"`
- Logs: `/root/.openclaw/workspace/chatbot/chatbot.log`

**Restart Server:**
```bash
cd /root/.openclaw/workspace/chatbot
pkill -f "node server.js"
npm start > chatbot.log 2>&1 &
```

**Check Status:**
```bash
curl http://localhost:3500/health
```

**View Logs:**
```bash
tail -f /root/.openclaw/workspace/chatbot/chatbot.log
```

**Start on Boot (systemd):**
```bash
sudo nano /etc/systemd/system/ffpma-chatbot.service
```

Paste:
```ini
[Unit]
Description=FFPMA Product Expert Chatbot
After=network.target

[Service]
Type=simple
User=root
WorkingDirectory=/root/.openclaw/workspace/chatbot
ExecStart=/usr/bin/npm start
Restart=always
RestartSec=10
Environment=NODE_ENV=production

[Install]
WantedBy=multi-user.target
```

Enable:
```bash
sudo systemctl daemon-reload
sudo systemctl enable ffpma-chatbot
sudo systemctl start ffpma-chatbot
```

## Troubleshooting

**Chat button doesn't appear:**
1. Check browser console for errors
2. Verify API URL is correct
3. Check if server is running: `curl http://130.49.160.73:3500/health`
4. Clear browser cache

**Chat doesn't respond:**
1. Check server logs: `tail -f chatbot.log`
2. Test API directly: `curl -X POST http://localhost:3500/api/chat -H "Content-Type: application/json" -d '{"message":"test"}'`
3. Verify OpenAI API key is valid
4. Check rate limits

**Products not loading:**
1. Test WooCommerce API: `curl https://www.forgottenformula.com/wp-json/wc/v3/products?consumer_key=XXX&consumer_secret=YYY`
2. Check WooCommerce API keys are valid
3. Restart server to force reload

**Slow responses:**
1. Check OpenAI API status
2. Reduce `max_tokens` in server.js
3. Consider caching common questions
